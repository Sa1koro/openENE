## 🔧 停用 Dell Precision 3630 的 BIOS 自动更新

### 根本原因
Windows Update 会把 Dell 的 BIOS/UEFI 固件当作「可选驱动」推送。BIOS 更新后通常会：
- 重置 BIOS 设置（如唤醒配置、启动顺序）
- 提示按 F1 确认配置变更
- **导致无人值守开机失败**

### ✅ 方法一：组策略/注册表关闭固件更新（推荐）

用这条命令就能一步到位禁用 Windows Update 推送固件和驱动：

```powershell
# 以管理员身份运行
reg add "HKLM\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate" /v ExcludeWUDriversInQualityUpdate /t REG_DWORD /d 1 /f
```

这会告诉 Windows Update：**不要包含驱动和固件到质量更新中**。BIOS、固件、驱动统统不推送。

> 你之前这条改过吗？目前看起来还没有设置过这个键值。现在设上就生效了，不需要重启。

### ✅ 方法二：Windows 更新设置里关「驱动更新」

如果你的 Windows 不是被组策略管理的（目前看起来你们有 WSUS），也可以在设置里关：

```
设置 → Windows 更新 → 高级选项
```

关掉 **"更新 Windows 时接收其他 Microsoft 产品的更新"**（收不到驱动和固件）。

### ✅ 方法三：在 BIOS 里关掉 UEFI Capsule 更新

这是更彻底的物理层阻止：

1. 开机按 F2 进 BIOS
2. 找 **System Configuration → UEFI Capsule Firmware Updates**
3. 设为 **Disabled**
4. 按下 F10 保存退出

> ⚠️ 这个设置要在机器前操作，如果你现在还没出差，赶紧设上。

### ✅ 方法四：隐藏已下载的 BIOS 更新

如果 Windows Update 已经下载了 BIOS 更新但还没安装：

```powershell
# 查看已下载的更新
Get-WUList | Where-Object {$_.Title -match "BIOS|System Firmware|Dell.*Firmware"}

# 隐藏特定的 BIOS 更新（使用 WU 的隐藏功能，需要 PSWindowsUpdate 模块）
```

更简单的方法：打开 Windows Update → **更新历史记录** → 卸载更新，找到 Firmware/Bios 更新卸载掉。

### ✅ 方法五：检查 WSUS 策略

你机器目前用的是 WSUS 更新服务器（`UseWUServer = 1`），那还需要检查**WSUS 服务器端**是否在推送固件。WSUS 管理员可以取消审批 BIOS 更新分类。

### 🛡️ 我建议你现在做三件事：

**立刻做的（无需重启）：**

```powershell

reg add "HKLM\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate" /v ExcludeWUDriversInQualityUpdate /t REG_DWORD /d 1 /f
```

- 确认 WSUS 管理控制台中 **Firmware 分类** 没有被批准推送到你的机器组
